Flow

Data + API paths

Targets

Single source of truth

  • PING_TARGET_URLS defines all monitored URLs.
  • Used by ping-agent and api-gateway.
  • Default: google.com, github.com.
Endpoints

API surface

  • GET /healthz
  • GET /targets
  • GET /uptime-summary
  • GET /uptime-summary-windowed?window=5m
  • GET /metrics
Metrics

Prometheus signals

  • ping_success_total and ping_failure_total
  • ping_latency_seconds histogram
  • api_gateway_requests_total
  • api_gateway_request_duration_seconds
Runtime knobs

Ping agent controls

  • PING_INTERVAL_SECONDS
  • PING_CONCURRENCY
  • PING_BODY_MAX_BYTES
  • PING_HTTP_METHOD
  • PING_RANGE_REQUEST
Retention

Prometheus storage

  • 14d retention
  • 5Gi PVC default size
  • Recreate strategy for PVC-bound pods
Alerting

TargetDown rule

  • Increase in ping_failure_total triggers alert
  • Alertmanager routes to alert-logger
  • SMTP credentials via values override
Quick checks

Endpoints to verify

  • curl http://localhost:8081/healthz
  • curl http://localhost:8081/uptime-summary
  • curl http://localhost:9090/targets
  • kubectl port-forward svc/iyup-ping-agent 18080:8080
  • kubectl port-forward svc/iyup-grafana 3000:3000
